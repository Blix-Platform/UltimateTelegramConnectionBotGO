## 📋 Описание
Telegram-бот на Go для двусторонней связи с поддержкой:
- 📨 Пересылки сообщений пользователей администратору
- 📩 Ответов администратора через систему reply
- 📁 Полной поддержки всех типов медиафайлов
- 🔨 Системы блокировки/разблокировки пользователей
- ⚙️ Гибкой настройки всех сообщений через `/settings`
- 🗃 Хранения данных в SQLite

**Технологии**:
- Go 1.16+
- `go-telegram-bot-api/v5`
- `SQLite` — надёжное встраиваемое хранилище
- Потокобезопасная работа через мьютексы

## 🚀 Установка

### Установка одной командой

#### Linux
```bash
curl -fsSL https://raw.githubusercontent.com/Blix-Platform/UltimateTelegramConnectionBotGO/main/install.sh -o install.sh && sudo bash install.sh
```

Или если репозиторий уже клонирован:
```bash
sudo bash install.sh
```

#### Windows
Запустите `install.bat` от имени администратора

Скрипт установки:
- Проверит и установит зависимости (Go, build-essential, gcc, libsqlite3-dev)
- Запросит токен бота и admin ID в красивом CLI интерфейсе
- Соберёт проект
- Создаст сервис с автоперезагрузкой

### Ручная установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Blix-Platform/UltimateTelegramConnectionBotGO.git
   cd UltimateTelegramConnectionBotGO
   ```

2. Скачайте зависимости:
   ```bash
   go mod download
   ```

3. Соберите и запустите:
   ```bash
   go build -o tgbot ./cmd/bot/

   # Linux/macOS
   export BOT_TOKEN="ваш_токен"
   export ADMIN_ID="ваш_id"
   ./tgbot

   # Windows
   set BOT_TOKEN=ваш_токен
   set ADMIN_ID=ваш_id
   tgbot.exe
   ```

## 📦 Версионирование и обновления

Бот использует **систему коммитов GitHub** с префиксом `[UP]`. Каждый коммит, содержащий `[UP]` в названии, считается обновлением.

### Команды

**`/version`** — показать теку версию, коммит и проверить доступность обновлений

**`/update`** — скачать и установить последние коммиты с `[UP]` через интерфейс бота

## ⚙️ Настройки бота (/settings)

Команда `/settings` открывает интерактивное меню с разделами для настройки всех текстовых сообщений бота.

### Как это работает

1. Администратор пишет `/settings`
2. Появляется главное меню с разделами: **💬 Сообщения** и **🔄 Обновить систему**
3. В разделе **💬 Сообщения** выбираете категорию:
   - 🛡️ **Админские сообщения** — ответы админа, подсказки, ошибки
   - 👤 **Пользовательские сообщения** — приветствие, запросы, подтверждения
   - 🚫 **Блокировки** — бан/анбан, уведомления о блокировке
4. Нажимаете на нужное сообщение
5. Показывается текущее значение и индикатор наличия фото (📷)
6. Отправляете **текст** или **фото с подписью** — настройка мгновенно сохраняется
7. Возврат в категорию через кнопку «⬅️ Назад к категориям»

### Поддержка фото

К любому сообщению можно прикрепить **фото**. Для этого при редактировании отправьте **фото с подписью**. Фото сохраняется и будет отправляться вместе с текстом при каждом вызове этого сообщения.

> 💡 Если при редактировании отправить только текст (без фото), ранее прикреплённое фото **сохранится**.

### Кнопка-ссылка

К любому сообщению можно добавить **инлайн-кнопку** с ссылкой:

1. В редакторе нажмите **«➕ Добавить кнопку-ссылку»**
2. Введите название кнопки
3. Введите URL
4. Кнопка появится под сообщением

Для удаления нажмите **«🔘 Удалить кнопку»** в редакторе.

### Форматирование

Все сообщения поддерживают **HTML-разметку**. При редактировании вы можете:

1. **Выделить текст** → меню Telegram → **Bold / Italic / Underline / Strikethrough / Monospace**
2. **Создать ссылку** → выделите текст → меню → **Create Link** → введите URL
3. **Ввести HTML вручную** → `<b>жирный</b>`, `<i>курсив</i>`, `<a href="url">ссылка</a>`

Telegram автоматически конвертирует форматирование в HTML при отправке сообщения, а бот сохраняет его в правильном виде.

| Формат | Telegram меню | HTML |
|--------|---------------|------|
| **Жирный** | Выделить → Bold | `<b>текст</b>` |
| *Курсив* | Выделить → Italic | `<i>текст</i>` |
| <u>Подчёркнутый</u> | Выделить → Underline | `<u>текст</u>` |
| ~~Зачёркнутый~~ | Выделить → Strikethrough | `<s>текст</s>` |
| `Код` | Выделить → Monospace | `<code>текст</code>` |
| [Ссылка](https://example.com) | Выделить → Create Link | `<a href="url">текст</a>` |

В редакторе сообщений предпросмотр отображается **с реальным форматированием** — жирный текст будет жирным, курсив — курсивом и т.д. Если к сообщению прикреплена фото, она также отобразится в редакторе.

## 🔑 Ключи настроек

Все ключи используются в команде `/resetms <ключ>` для сброса к значению по умолчанию, а также редактируются через `/settings`.

### 🛡️ Админские сообщения

| Ключ | Описание | Значение по умолчанию |
|------|----------|----------------------|
| `otvet_msg` | Префикс ответа админа | Ответ от администратора: |
| `areply_msg` | Админу об отправке ответа | Ваш ответ успешно отправлен пользователю! |
| `aeror_msg` | Ошибка получателя | Не удалось определить получателя. Возможно, сообщение устарело. |
| `ar_msg` | Подсказка про reply | Пожалуйста, ответьте на сообщение пользователя, чтобы отправить ему ответ. |

### 👤 Пользовательские сообщения

| Ключ | Описание | Значение по умолчанию |
|------|----------|----------------------|
| `start_msg` | Приветствие (/start) | Привет! Это бот для отправки предложений. Напишите ваше сообщение, и оно будет доставлено администратору. |
| `take_msg` | Запрос на отправку сообщения | Пожалуйста, отправьте текстовое сообщение или медиафайл. |
| `gift_msg` | Успешная отправка текста | Ваше сообщение успешно отправлено администратору! |
| `fgift_msg` | Успешная отправка файла | Ваш файл успешно отправлен администратору! |

### 🚫 Блокировки

| Ключ | Описание | Значение по умолчанию |
|------|----------|----------------------|
| `ban_usage_msg` | Инструкция /ban | Использование: /ban <user_id> <часы> [причина]\nИли ответьте на сообщение пользователя: /ban <часы> [причина] |
| `ban_success_msg` | Бан без причины | Пользователь {user_id} заблокирован на {hours} ч. |
| `ban_success_reason_msg` | Бан с причиной | Пользователь {user_id} заблокирован на {hours} ч. Причина: {reason} |
| `ban_no_reason_msg` | Если причина не указана | Причина не указана. |
| `banned_msg` | Сообщение заблокированному | Вы заблокированы.\nОсталось времени: {time}\nПричина: {reason} |
| `unban_success_msg` | Успешный /unban | ✅ Пользователь {user_id} разблокирован |
| `unban_usage_msg` | Инструкция /unban | Использование: /unban <user_id>\nИли ответьте на сообщение пользователя: /unban |

### 🔄 Плейсхолдеры

В некоторых настройках работают специальные плейсхолдеры, которые автоматически заменяются:

#### Для `banned_msg`:
| Плейсхолдер | Описание |
|-------------|----------|
| `{time}` | Оставшееся время блокировки (автоформат: часы + минуты) |
| `{reason}` | Причина блокировки |

#### Для `ban_success_msg` и `ban_success_reason_msg`:
| Плейсхолдер | Описание |
|-------------|----------|
| `{user_id}` | ID заблокированного пользователя |
| `{hours}` | Время блокировки в часах |
| `{reason}` | Причина блокировки |

## 🔧 Команды бота

### 👤 Команды пользователей

#### `/start`

Начало работы с ботом. Отправляет приветственное сообщение, настраиваемое через `start_msg`.

---

### 🛡️ Команды администратора

Все команды ниже доступны только администратору.

---

#### 🔨 `/ban`

Блокирует пользователя на определённое время.

**Использование:**

С указанием ID пользователя:
```
/ban <user_id> <часы> [причина]
```

Ответом на сообщение (без ID):
```
/ban <часы> [причина]
```

**Примеры:**
```
/ban 123456789 24 Оскорбления
/ban 48 Спам
/ban 1.5 Флуд
```

**Параметры:**
| Параметр | Описание |
|----------|----------|
| `user_id` | Telegram ID пользователя (не требуется при ответе на сообщение) |
| `часы` | Время блокировки в часах (поддерживаются дробные числа) |
| `причина` | Необязательный параметр |

> ⚠️ Администратор и сам бот **не могут быть заблокированы** — бот автоматически отклонит такую команду.

---

#### 🔓 `/unban`

Разблокирует пользователя.

**Использование:**
```
/unban <user_id>
```

Или **ответом на сообщение пользователя**:
```
/unban
```

**Пример:**
```
/unban 123456789
```

---

#### 🔄 `/resetsettings`

Проверяет и восстанавливает недостающие настройки по умолчанию.

**Когда использовать:**
- После обновления бота появились новые настройки, которых нет в старой базе
- Некоторые кнопки в `/settings` не работают (пустые значения)

**Использование:** просто напишите `/resetsettings` в чате с ботом (только администратор)

> Команда **не перезаписывает** существующие настройки, только добавляет недостающие.

---

#### 🔑 `/resetms`

Сбрасывает конкретную настройку к значению по умолчанию, включая удаление прикреплённого фото и кнопки-ссылки.

**Использование:**
```
/resetms <ключ>
```

**Пример:**
```
/resetms start_msg
/resetms banned_msg
```

> ⚠️ Эта команда **полностью очищает** настройку: текст, фото и кнопку-ссылку, затем восстанавливает значение по умолчанию.

---

#### 🔄 `/update`

Автоматическое обновление бота из **GitHub Releases** без потери данных.

**Как работает:**
1. Проверяет последний релиз через GitHub API
2. Если версия совпадает — сообщает «У вас последняя версия»
3. Если есть обновление — скачивает архив релиза
4. Сохраняет `.env` и `bot.db`
5. Заменяет файлы, собирает, перезапускает
6. Пишет: «Система обновлена!»

**Использование:** просто напишите `/update` в чате с ботом (только администратор)

> ⚠️ Если версия уже последняя, обновление не будет загружено

Также обновление доступно через `/settings → 🔄 Обновить систему → ⬇️ Обновить`.

---

#### 📦 `/version`

Показывает теку версию, коммит и проверяет доступность обновлений.

**Использование:** просто напишите `/version` в чате с ботом (только администратор)

## ❌ Ошибка авторизации

Если в логах появляется ошибка `SESSION_REVOKED` или `Unauthorized`:

1. Токен бота был отозван через @BotFather
2. Обновите токен:
   ```bash
   sudo nano /opt/tgbot/.env
   # Укажите новый BOT_TOKEN=ваш_новый_токен
   ```
3. Перезапустите сервис:
   ```bash
   sudo systemctl restart tgbot
   ```

## 🗑 Удаление

### Linux (одной командой)
```bash
curl -fsSL https://raw.githubusercontent.com/Blix-Platform/UltimateTelegramConnectionBotGO/main/uninstall.sh -o uninstall.sh && sudo bash uninstall.sh
```

Или из директории проекта:
```bash
sudo bash uninstall.sh
```

### Windows
Запустите `uninstall.bat`

Скрипт удаления:
- Остановит и удалит сервис
- Удалит директорию `/opt/tgbot` (Linux) или `tgbot\` (Windows)
- Удалит базу данных SQLite и все настройки
- Очистит логи

> ⚠️ **Все данные будут удалены без возможности восстановления**

## 📂 После установки

**Файлы:**
- `.env` — токен и admin ID (Linux: `/opt/tgbot/.env`)
- `bot.db` — SQLite база с настройками, банами и соответствиями сообщений

```bash
systemctl status tgbot       # статус
systemctl restart tgbot      # перезапуск
systemctl stop tgbot         # остановка
journalctl -u tgbot -f       # логи
```

- Запуск: ярлык `StartBot.lnk`
- Остановка: закройте окно терминала
- Автозапуск: добавлен в Планировщик заданий
